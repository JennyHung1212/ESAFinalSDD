Chunked Design

Subsystems

Randomization Subsystem: Randomly deals cards, for use in the penalty stage Flow Subsystem:
Ensures rules are followed and facilitates smooth game flow		 Timer Subsystem:
Determines who the last player to execute the action in the playing stage is		 Card Recognition and Tracking Subsystem:
Identifies the type of playing cards flipped open in the playing stage and records the movement and quantity of cards
The system should have random elements that reduce the chances of a player monopolizing the game The system should be able to resolve ties in the playing stage The system should be able to detect who the last person to execute an action in the playing stage is The system should be able to track the number of playing cards on hand
The system should be able to randomly deal a penalty card to penalized player The system should allow game continuation when the penalty cards run out The system should be able to check if a 'local' playing card has been played
Challenge penalty cards The system should penalise people who break the rules The system should be able to check if a match between 'object' playing cards has been made
Fate penalty cards The system should be able to record the number of penalty cards each player has
The system should be able to check when a penalty card has been kept by a player
